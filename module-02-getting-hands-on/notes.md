# Модуль 2 — Getting Hands On
На початку кожного нового проекту треба написати `/init` — Claude сам читає всі файли і створює `CLAUDE.md` де описує проект, як його запускати і що важливо знати. Таких файлів може бути три: головний який я комічу в git і всі бачать, локальний тільки для себе, і глобальний який діє на всі мої проекти на машині.
Щоб додати якесь правило просто написати `#`, наприклад `# завжди пиши коментарі українською`. Дуже зручно, якщо він постійно робить одну й ту саму помилку.
`@` використовується, щоб вказати конкретний файл в промпті.
Planning Mode вмикається через `Shift+Tab` двічі. Тоді Claude спочатку читає весь проект, складає план і показує його перед тим як щось робити. Thinking Modes — рівні глибини міркування: `think`, `think more`, або `ultrathink` для найскладнішого. Planning це широко по проекту, thinking це глибоко по конкретній задачі, і їх можна поєднувати.
`Escape` зупиняє Claude. Подвійний `Escape` показує всі попередні повідомлення і я можу повернутись до будь-якої точки розмови.
`/compact` стискає розмову але зберігає все що він дізнався про мій проект, робить підсумок всіх попередніх дій. `/clear` повний скид.
Щоб зробити свої команди створюємо `.md` файл у папку `.claude/commands/` і ім'я файлу стає командою. Всередині файлу пишемо, що Claude має зробити коли викликаємо цю команду. Можна зробити команду з аргументом через `$ARGUMENTS`. Після додавання нових команд треба перезапустити Claude.
MCP — це спосіб додати Claude нові інструменти яких у нього нема за замовчуванням.Playwright дає йому можливість керувати браузером: відкривати сторінки, клікати, заповнювати форми і тд.
Щоб підключити Claude до GitHub: `/install-github-app`.

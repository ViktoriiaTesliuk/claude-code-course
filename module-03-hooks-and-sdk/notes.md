# Модуль 3 — Hooks and the SDK

Hooks – це можливість запускати свій код до або після того як Claude щось робить. Наприклад, автоматично форматувати файл після кожного редагування або блокувати доступ до `.env`.
`PreToolUse` запускається до того як Claude виконує дію і може її заблокувати. `PostToolUse` запускається після і вже не може заблокувати, але може запустити щось додатково або дати зворотній зв'язок.
Хуки можна налаштувати на трьох рівнях: глобально для всіх проектів, на рівні проекту (комітиться в git), або локально тільки для себе. Налаштовуються в `settings.json`, або можна зробити інтерактивно через `/hooks` всередині Claude Code.
Щоб написати хук треба вибрати тип (`PreToolUse` або `PostToolUse`), вказати matcher (який інструмент відслідковувати), написати скрипт, і повернути правильний exit code. `0` означає дозволити, `2` означає заблокувати. Все що пишеш в `stderr` Claude читає як пояснення чому заблоковано.
Коли хук запускається, він отримує JSON через stdin з інформацією про те що Claude хоче зробити (назву інструменту, шлях до файлу або команду). Скрипт це парсить і вирішує що робити.
Крім `PreToolUse` і `PostToolUse` є ще інші типи: `Stop` коли Claude закінчив відповідати, `SessionStart` і `SessionEnd` на початку і кінці сесії, `UserPromptSubmit` коли я відправляю промпт, і ще кілька. Щоб зрозуміти який JSON приходить в хук можна зробити дебаг-хук який просто пише всі дані в файл через `jq`.
SDK – це можливість запускати Claude Code програмно зі своїх скриптів, CI/CD пайплайнів або інших інструментів. Є версії для TypeScript, Python і CLI. За замовчуванням SDK має тільки права на читання, щоб дати права на запис треба явно вказати `allowedTools`. Використовувати можна для автоматичного review перед комітом, генерації документації, перевірки якості коду в пайплайні і тд.
